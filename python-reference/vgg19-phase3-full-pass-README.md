# VGG19 phase-3 full-pass fixture generator

This document describes the large fixture exporter for full VGG19 forward/loss parity through `conv5_1`.

## Generate

From the repository root:

```bash
python python-reference/export_vgg19_phase3_full_pass.py
```

## Output directory

The script writes to:

```text
public/vgg19-phase3-full-pass/
```

## Exported files

- `vgg19_conv0_to_conv28_weights.json`
  - Legacy JSON weights and biases for VGG19 convolution layers encountered through feature layer index 28.
- `vgg19_phase3_full_pass_fixture.json`
  - 16x16 content/style/input tensors.
  - Normalization mean/std values.
  - Style tap indices and expected style losses.
  - Content tap index and expected content loss.
  - Expected total loss values.
  - Gradient fields when generated by the exporter version in use.

## Consumers

- `tests/phase3-full-pass.spec.ts`
- `tests/phase5-full-style-transfer-optimization.spec.ts`
- `tests/kernel-lab.spec.ts`
- `/benchmark` full-style and kernel-lab flows

Some consumers also require a manifest-backed `fp32` model pack under `public/vgg19-models/fp32/` or an external model base URL that provides it.

## Commit policy

The generated full-pass artifacts are intentionally large and are not committed by default. Generate them locally when running optional full-pass parity or benchmark tests, and do not commit newly generated artifacts unless the task explicitly requires it.
